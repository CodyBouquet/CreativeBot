GOAL
----
Add a second tool to the Pi alongside the existing Pipedrive/Arrivy
sync. This one pulls data from the Rollmaster API and emails scheduled
reports to Entra (Azure AD) group members via Microsoft Graph.

On sign-in the user should see a landing page with two action cards:
  - "Pipedrive / Arrivy Sync"  (the existing tool, moves to /sync)
  - "Email Reports"            (the new tool, lives at /reports)


REPORTS WANTED (initial list, more to come)
-------------------------------------------
- Low inventory notification
- Daily sales report    (overall, by salesperson, by branch)
- Weekly sales report   (overall, by salesperson, by branch)
- Monthly sales report  (overall, by salesperson, by branch)

Each report needs:
  - A configurable frequency (daily/weekly/monthly)
  - A configurable send time
  - A target Entra group (recipients resolved at send time)


ARCHITECTURE DECISIONS
----------------------
1. ONE Flask app, TWO tools as blueprints.
   - New file: email_reports.py, registered as a Flask blueprint
   - Shares the existing PIN login
   - "Completely separate" in code, DB tables, and UI — but not a
     separate service. One deploy, one login.

2. Landing page at /
   - Two action cards
   - Current dashboard moves from / to /sync

3. Separate SQLite DB for the new tool: data/reports.db
   - Tables: report_subscriptions, send_history, settings
   - Keeps the two tools isolated — can't corrupt each other

4. Email via Microsoft Graph API (NOT SMTP)
   - Graph is the modern M365 path
   - Same Azure app registration handles mail send + group lookup
   - SMTP would be simpler but can't list group members

5. Recipients = Entra groups, NOT stored email addresses
   - User picks a group in the UI per report
   - At send time: Graph resolves "who's in this group?" and emails each
   - Adding/removing people happens in Entra, Pi picks up automatically
   - No recipient management code in the Pi at all

6. Sender = a shared mailbox (recommended: noreply@creativecarpetinc.com)
   - M365 shared mailboxes are free (no license needed)
   - Create once in Exchange admin center
   - Graph Mail.Send can send as it
